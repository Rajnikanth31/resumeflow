# Support Guide

This document defines the official support channels and procedures for the ResumeFlow project.

---

## 1. Asking Questions

For usage questions, custom configuration help, or troubleshooting:

- **GitHub Discussions**: We use GitHub Discussions for all community Q&A. Please check existing threads or open a new discussion under the **Q&A** category.
- **Documentation**: Verify setup rules in `README.md` and `DOCKER.md`.

---

## 2. Reporting Bugs

If you identify a bug or regression in the codebase:

1.  Verify the issue persists on the latest `main` branch build.
2.  Search open GitHub Issues to ensure it hasn't already been reported.
3.  Open a new issue using our **🐛 Bug Report** template, providing reproduction steps and OS/browser environments.

---

## 3. Requesting Features

To propose enhancements or new features:

1.  Check the roadmap epics in `docs/project-management/milestones.md` to see if it is already planned.
2.  Open a GitHub Issue using the **🚀 Feature Request** template.

---

## 4. Security Concerns

For reporting security vulnerabilities, please do not open a public issue. Follow our security disclosure protocol outlined in `SECURITY.md` by emailing `security@resumeflow.com`.
